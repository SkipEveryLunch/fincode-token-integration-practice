# A.　現実装での購入時の流れ

1. フロントエンドの購入ボタンを押下
2. DBに保存されたcustomer_id, card_idを用いてサーバー側で決済を作成->実行。
   - 3DSチャレンジが必要ならフロント側にredirect_urlを返す
3. フロントエンドで、3DSのチャレンジ画面にリダイレクトする
4. 3DSチャレンジに成功すると、サーバー側のコールバックパスにリダイレクトする
   - 必要ならここで何らかの処理を行うこともできるが、現在はフロントへのリダイレクトのみを行なっている
5. フロントエンドで「購入成功」と表示

# B. 2026-05-17 時点での問題

現時点ではwebhookのトリガには「決済実行」イベントを使っている。
つまり3DSチャレンジ以前の段階（A-2）でwebhookにリクエストが送られる。
この時点でpaymentのstatusを書き換えると、その後の3DSチャレンジの結果を無視することになる。

しかし「3DSチャレンジ　結果確定」をトリガイベントとして選ぶと、今度は3DSチャレンジが発生しなかった場合にトリガされない。
また「3DSチャレンジ　結果確定」のあとに考慮すべきイベントが本当にないのかという疑問がある（3DSチャレンジには成功したが残高が足りないケースなど）。

「売上確定」イベントを選択すべきとも思えるが、なぜかこのイベントはAのフローではトリガされない。

# C. Bの解決

「カード決済 認証後決済実行」イベントを選ぶことで、以下の両方でリクエストが飛んでくることがわかった。

- 3DSチャレンジが発生する場合
- 3DSチャレンジが発生しない場合
